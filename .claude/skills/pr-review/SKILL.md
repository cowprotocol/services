---
name: pr-review
description: How to review a pull request — keep the overall summary short and skip nits.
---

When reviewing a pull request:

- Keep the overall summary comment short — a few lines at most: overall assessment plus a count of issues by severity. Don't restate the per-line findings there.
- Don't comment on nits, style preferences, or minor observations. Only flag issues that could cause incorrect behaviour, a bug, a security or performance problem, or a test failure. If there's nothing substantive to flag, say so in one line.
